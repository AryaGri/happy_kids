### 3.1 Внешние ключи (ForeignKey, OneToOne)

| Связь | Тип | Кардинальность | Направление | Ограничения | Описание |
|-------|-----|----------------|-------------|-------------|----------|

### 3.2 Связи ManyToMany (без through)

| Связь | Тип | Кардинальность | Направление | Ограничения | Описание |
|-------|-----|----------------|-------------|-------------|----------|
| CUsers.children | M2M (self) | M:N | CUsers ⟷ CUsers | symmetrical=False, blank=True, related_name='parents' | Родитель ↔ Дети. Сторона «многие»: родитель имеет много детей; ребёнок имеет много родителей |

## 4. Таблицы-связки ManyToMany (автоматически созданные Django)

Django не использует явную модель `through` для указанных M2M — создаются автоматические промежуточные таблицы.

### 4.1 accounts_cusers_children

**Назначение:** связь родитель–ребёнок (CUsers.children / CUsers.parents).

| Столбец | Тип | Ограничения | Описание |
|--------|-----|-------------|----------|
| id | INTEGER | PK, auto | Первичный ключ |
| from_cusers_id | INTEGER | FK → accounts_cusers.id, NOT NULL | ID пользователя-родителя (источник связи) |
| to_cusers_id | INTEGER | FK → accounts_cusers.id, NOT NULL | ID пользователя-ребёнка (целевой объект) |

**Уникальность:** Django создаёт ограничение UNIQUE (from_cusers_id, to_cusers_id) для пар.

**Примечание:** для self-referential M2M Django использует имена `from_<model>_id` и `to_<model>_id`.

### 4.2 accounts_cusers_patients

**Назначение:** связь врач–пациенты (CUsers.patients / CUsers.doctors).

| Столбец | Тип | Ограничения | Описание |
|--------|-----|-------------|----------|
| id | INTEGER | PK, auto | Первичный ключ |
| from_cusers_id | INTEGER | FK → accounts_cusers.id, NOT NULL | ID врача (источник) |
| to_cusers_id | INTEGER | FK → accounts_cusers.id, NOT NULL | ID ребёнка-пациента (целевой) |

**Уникальность:** UNIQUE (from_cusers_id, to_cusers_id).

### 4.3 accounts_diagnosticprofile_based_on_sessions

**Назначение:** связь DiagnosticProfile ⟷ GameSession (based_on_sessions / profiles).

| Столбец | Тип | Ограничения | Описание |
|--------|-----|-------------|----------|
| id | INTEGER | PK, auto | Первичный ключ |
| diagnosticprofile_id | INTEGER | FK → accounts_diagnosticprofile.id, NOT NULL | ID профиля |
| gamesession_id | INTEGER | FK → accounts_gamesession.id, NOT NULL | ID сессии |

**Уникальность:** UNIQUE (diagnosticprofile_id, gamesession_id).

## 7. Резюме для ER-диаграммы

### 7.1 Центральная сущность

**CUsers** — единая модель пользователей с ролями (admin, doctor, parent, child). Первичный ключ: id.

### 7.2 Иерархия и связи

```
CUsers (1) ────┬── (1:1) ──── DoctorLicense
               ├── (1:1) ──── Subscription
               ├── (1:N) ──── GameSession
               │                └── (1:N) ──── GameResult
               ├── (1:N) ──── GameResult (напрямую)
               ├── (1:N) ──── Prescription (как child)
               ├── (1:N) ──── Prescription (как doctor)
               ├── (1:N) ──── DiagnosticProfile
               ├── (1:N) ──── ApiToken
               ├── (1:N) ──── DoctorLicense.verified_by
               ├── (M:N) ──── CUsers (children ⟷ parents)
               └── (M:N) ──── CUsers (patients ⟷ doctors)

DiagnosticProfile (M:N) ⟷ GameSession (через accounts_diagnosticprofile_based_on_sessions)

FuzzyLinguisticVariable (1) ──── (1:N) ──── FuzzyMembershipFunction

DiagnosticDiagnosis, FuzzyInferenceRule, BehaviorPattern — изолированные справочники
```

### 7.3 Ключевые сущности для диаграммы

1. **CUsers** — пользователи (4 роли).
2. **DoctorLicense** — лицензия врача (1:1 с CUsers).
3. **GameSession** — игровая сессия (N:1 к CUsers).
4. **GameResult** — результат игры (N:1 к CUsers, N:1 к GameSession, nullable).
5. **Prescription** — назначение (N:1 к CUsers как child, N:1 к CUsers как doctor, doctor nullable).
6. **DiagnosticProfile** — профиль ребёнка (N:1 к CUsers, M:N с GameSession).
7. **Subscription** — подписка (1:1 с CUsers).
8. **ApiToken** — токен API (N:1 к CUsers).
9. **FuzzyLinguisticVariable** — лингвистическая переменная.
10. **FuzzyMembershipFunction** — функция принадлежности (N:1 к FuzzyLinguisticVariable).
11. **DiagnosticDiagnosis**, **FuzzyInferenceRule**, **BehaviorPattern** — справочники без FK.

### 7.4 Промежуточные таблицы M2M

- `accounts_cusers_children` — родитель ↔ ребёнок.
- `accounts_cusers_patients` — врач ↔ пациент (ребёнок).
- `accounts_diagnosticprofile_based_on_sessions` — профиль ↔ игровые сессии.

### 7.5 Ограничения целостности

- **CASCADE:** DoctorLicense, GameSession, GameResult, Prescription, DiagnosticProfile, FuzzyMembershipFunction, Subscription, ApiToken — при удалении связанной записи удаляются зависимые.
- **SET_NULL:** DoctorLicense.verified_by — при удалении администратора поле обнуляется.
- **nullable FK:** GameResult.session, Prescription.doctor.
- **unique_together:** FuzzyMembershipFunction (method_class, variable).
